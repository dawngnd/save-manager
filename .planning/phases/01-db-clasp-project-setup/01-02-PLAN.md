# Plan 01-02: Configure clasp development environment and basic doGet test script

## Overview

Tạo file `.clasp.json` để kết nối local project với GAS cloud, file `.claspignore` để loại trừ file không cần push, và thêm function `doGet(e)` vào `Code.js` trả về JSON response để verify clasp push + deploy hoạt động.

- Tạo JSON object với các field:
  - `status`: `"ok"`
  - `message`: `"Save Manager backend is running"`

Function này dùng để verify sau khi `clasp push` + `clasp deploy` rằng backend hoạt động đúng và timezone trả về là `Asia/Ho_Chi_Minh`.

**Section "Verification: clasp Push & Deploy":**
1. Đảm bảo đã login: `clasp login`
2. Thay `YOUR_SCRIPT_ID_HERE` trong `.clasp.json` bằng Script ID thực (từ GAS Editor > Project Settings > IDs)
3. Push code: `cd backend && clasp push`
4. Deploy: `clasp deploy --description "Phase 1 test"`
5. Lấy deployment URL: `clasp deployments`
6. Test URL: Mở `{URL}/exec` trong browser, verify JSON response chứa `"status": "ok"` và `"timezone": "Asia/Ho_Chi_Minh"`
7. Hoặc dùng `{URL}/dev` để test development version (tránh caching)

**Lưu ý**: Luôn dùng `/dev` URL khi đang develop để tránh GAS caching response cũ.
</action>
