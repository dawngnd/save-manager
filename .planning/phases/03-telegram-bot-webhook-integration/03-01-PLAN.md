<objective>
Thiết lập kết nối Webhook giữa Telegram Bot API và GAS Web App, cập nhật cấu trúc cơ sở dữ liệu để lưu trữ Chat ID người dùng, xử lý tin nhắn /start để trả về nút mở Mini App, và tự động liên kết Chat ID khi người dùng tương tác qua Mini App.

Purpose: Đặt nền móng kết nối Telegram Bot, cho phép người dùng mở ứng dụng trực tiếp từ giao diện chat và thu thập thông tin định danh (Chat ID) để phục vụ tính năng gửi thông báo tự động.
Output: backend/Code.js (nâng cấp doPost, bổ sung webhook logic và DB helpers), backend/Tests.js (bổ sung unit test tích hợp).
</objective>

<task type="auto" tdd="true">
  <name>Task 1: Cập nhật cấu trúc bảng Users và bổ sung các hàm xử lý Webhook Telegram</name>
  <files>backend/Code.js</files>
  <behavior>
    - initializeSheets tạo bảng Users với header ['username_bankcode', 'telegram_chat_id'].
    - linkTelegramChatId ghi đè chat_id cũ nếu chat_id mới khác biệt, append dòng mới nếu user chưa tồn tại.
    - doPost từ chối webhook Telegram nếu token query parameter không khớp WEBHOOK_TOKEN.
    - handleTelegramWebhook phản hồi tin nhắn /start bằng tin nhắn chứa nút bấm mở Mini App tương ứng với MINI_APP_URL.
    - doPost action get_deposits tự động gọi linkTelegramChatId nếu payload chứa telegram_chat_id.
  </behavior>
  <action>
    Cập nhật backend/Code.js để bổ sung các tính năng tích hợp Telegram:
    
    1. Cập nhật `initializeSheets()` (theo 'Sheet initialization pattern' của 03-PATTERNS.md):
       - Đổi `usersHeaders` thành `['username_bankcode', 'telegram_chat_id']` để chuẩn bị cột lưu trữ.
    
    2. Triển khai `linkTelegramChatId(usersSheet, usernameBankcode, telegramChatId)` (theo 'Webhook Routing & Token Authorization pattern' của 03-PATTERNS.md):
       - Tìm kiếm dòng của `usernameBankcode` trong sheet Users.
       - Nếu tìm thấy, ghi đè giá trị `telegramChatId` tại cột thứ 2 nếu giá trị hiện tại khác biệt.
       - Nếu không tìm thấy, appendRow dòng mới gồm `[usernameBankcode, telegramChatId]`.
    
    3. Triển khai `setupWebhook()` (theo 'Properties & API Call Credentials' của 03-PATTERNS.md và D-04):
       - Lấy `TELEGRAM_BOT_TOKEN`, `WEB_APP_URL` và `WEBHOOK_TOKEN` từ Script Properties.
       - Gọi API `setWebhook` của Telegram bằng `UrlFetchApp.fetch`, truyền URL webhook có đính kèm tham số `?token={WEBHOOK_TOKEN}`.
       - Log kết quả đăng ký webhook.
    
    4. Triển khai `sendTelegramApi(method, payload)` (theo 'Properties & API Call Credentials' của 03-PATTERNS.md):
       - Gửi POST request đến `https://api.telegram.org/bot{TELEGRAM_BOT_TOKEN}/{method}` sử dụng `UrlFetchApp.fetch`.
       - Thiết lập `contentType` thành `application/json` và `payload` là chuỗi JSON. Bật `muteHttpExceptions`.
    
    5. Triển khai `handleTelegramWebhook(payload)` (theo 'Properties & API Call Credentials' của 03-PATTERNS.md và D-01):
       - Phân tích payload từ Telegram. Nếu tin nhắn có text `/start`:
         * Lấy `MINI_APP_URL` từ Script Properties.
         * Gửi tin nhắn chào mừng kèm nút Inline Keyboard loại `web_app` trỏ đến `MINI_APP_URL` thông qua `sendTelegramApi("sendMessage", payload)`.
       - Trả về JSON success báo webhook đã xử lý.
    
    6. Cập nhật `doPost(e)` (theo 'Webhook Routing & Token Authorization pattern' và 'Error Handling & Logging pattern' của 03-PATTERNS.md):
       - Đọc token query parameter: `e.parameter.token`.
       - Parse payload: `JSON.parse(e.postData.contents)`.
       - Nhận diện request từ Telegram: `payload.message || payload.callback_query`.
       - Nếu là Telegram webhook:
         * Xác thực token query parameter khớp với `WEBHOOK_TOKEN` lưu trong Script Properties. Trả về lỗi 401/403 nếu không khớp (per D-04).
         * Gọi `handleTelegramWebhook(payload)`.
       - Nếu là request bình thường từ Frontend:
         * Trong nhánh `action === "get_deposits"`: Nếu payload chứa `telegram_chat_id`, gọi `linkTelegramChatId` để tự động liên kết tài khoản.
  </action>
  <verify>
    <automated>grep -E "function linkTelegramChatId|function setupWebhook|function sendTelegramApi|function handleTelegramWebhook" backend/Code.js</automated>
  </verify>
  <done>Mã nguồn Code.js chứa đầy đủ các định nghĩa hàm tích hợp Webhook và lưu trữ Chat ID hoạt động chính xác.</done>
</task>
