<task type="auto" tdd="true">
  <name>Task 1: Triển khai hàm doPost và điều phối nghiệp vụ ghi bằng LockService</name>
  <files>backend/Code.js</files>
  <behavior>
    - Yêu cầu POST với action không hợp lệ trả về JSON status: error.
    - Yêu cầu POST action get_deposits chạy bình thường không dùng lock.
    - Yêu cầu POST action add_deposit hoặc rollover_deposit thực thi lock.tryLock(10000) bảo vệ ghi.
    - Nếu không lấy được lock, trả về status: error và message chứa "Hệ thống đang bận, vui lòng thử lại.".
    - Kết quả trả về từ doPost luôn có định dạng ContentService.MimeType.JSON.
  </behavior>
  <action>
    Bổ sung và tích hợp các hàm API Controller vào backend/Code.js (tham chiếu cấu trúc và ví dụ từ analog backend/Code.js trong 02-PATTERNS.md):
    
    1. doPost(e):
       - Nhận sự kiện POST và kiểm tra dữ liệu đầu vào.
       - Parse nội dung JSON trong e.postData.contents để lấy thuộc tính `action` (per D-01).
       - Định tuyến yêu cầu:
         * action "get_deposits": gọi executeGetDeposits(sheets, payload) không sử dụng lock.
         * action "add_deposit" hoặc "rollover_deposit": chuyển qua bộ quản lý lock handleWriteActionWithLock.
         * Khác: trả về lỗi không hỗ trợ hành động.
    
    2. handleWriteActionWithLock(action, payload):
       - Gọi LockService.getScriptLock() lấy đối tượng Script Lock của Google Apps Script (per D-03, DB-02).
       - Thử khóa ghi bằng lock.tryLock(10000) với timeout 10 giây (per D-03).
       - Nếu không lấy được khóa, trả về JSON thông báo lỗi bận hệ thống (per D-03).
       - Nếu lấy được khóa, gọi initializeSheets() khởi tạo database rồi thực hiện nghiệp vụ executeAddDeposit hoặc executeRolloverDeposit tương ứng.
       - Sử dụng cấu trúc try-catch-finally để đảm bảo lock.releaseLock() luôn được gọi giải phóng khóa an toàn khi kết thúc thao tác (hoặc khi gặp crash).
    
    3. buildJsonResponse(status, messageOrData):
       - Đóng gói dữ liệu đầu ra thành JSON object `{ status: status, data: ... }` hoặc `{ status: status, message: ... }`.
       - Tạo và thiết lập MIME Type ContentService.MimeType.JSON cho Content Service output.
  </action>
  <verify>
    <automated>grep -E "function doPost|function handleWriteActionWithLock|function buildJsonResponse" backend/Code.js</automated>
  </verify>
  <done>Hàm doPost, handleWriteActionWithLock và buildJsonResponse được định nghĩa đầy đủ và tuân thủ các quyết định API thiết kế.</done>
</task>
